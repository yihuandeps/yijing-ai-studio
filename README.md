# 衣境 AI · 服饰创作台

面向服饰商家的 AI 图片 / 视频生成工具 —— **LUI 对话式布局**的高保真交互原型。

**🔗 在线体验:<https://yihuandeps.github.io/yijing-ai-studio/>**

## 使用方式

双击打开 `index.html` 即可在浏览器中预览（零依赖、零构建，需联网加载字体与示例图片）。

## 文件说明

| 文件 | 说明 |
|------|------|
| `index.html` | 完整交互原型（HTML + CSS + 原生 JS 单文件） |
| `DESIGN.md` | 设计规范：色板 / 字体 / 组件 / 动效 / 响应式 / 设计护栏，代码严格依此生成 |

## 核心组成（对应产品五大模块）

1. **对话创作流 + 图片上传** — 中央 LUI 主区：欢迎态快捷创作卡、输入框支持点击 / 拖拽 / 粘贴上传，图片-视频模式切换、模特定制（跟随风格推荐 / 4 个预设数字模特 / 年龄×长相自主组合（长相含清纯邻家、高级模特脸等 6 款，未选维度自动跟随风格）/ 自定义外貌描述）、比例与清晰度（1K/2K/4K）选择、Enter 发送，含模拟的流式回复与生成进度、结果预览 / 下载 / 重新生成 / 转视频
2. **商家资产库** — 右侧面板页签一：商品图 / 模特图 / 生成结果分类管理，支持上传，生成结果一键归档
3. **商家场景库** — 页签二：门店实景、品牌棚景，可上传图片新建自定义场景
4. **公共场景库** — 页签三：平台精选场景（街拍 / 居家 / 度假 / 商业），点「使用」即设为生成场景
5. **风格提示词库** — 页签四：19 个女装风格 · 21 条专业生图提示词（模块库 20260719-2124 版，由 `拼装库工程/export_style_lib.py` 按拼装器默认配方生成：半身图替代特写、默认无发饰约束、全身自动附加入画约束），按大类筛选，点「全身主图 / 半身图」插入输入框并自动选中【服装描述】占位；提示词中的画幅与模特段均为动态占位（不写死）——画幅按所选比例填充、模特按「模特」选择器填充（跟随风格推荐 / 预设 / 年龄×长相组合 / 自定义），切换比例或模特都会实时同步已插入的提示词；清晰度由独立选择器控制、不进提示词文本，卡片角标提示该风格推荐画幅。**所有选项均可见提示词**：风格卡「查看提示词」按当前比例/模特实时填充展开全文，预设模特直接显示片段、年龄长相悬停显示文案，模特弹窗底部有固定的「当前模特提示词」实时预览条，比例选项标注对应画幅短语

6. **风格提示词拼装器**（注入模块，`拼装库工程/integrate_assembler.py` 同步）— 工具栏魔杖按钮 / 欢迎快捷卡打开：19 风格点选成词，文生图/图生图 × 全身/半身 × 实景/影棚 × 版型（跟随/修身/宽松）× 约束（头发无饰品，默认开），动作槽含 23 条在用通用动作（G-动作1~27，弃用自动出池）、发妆槽可选无发饰版；全身自动附加入画约束，带发饰残留 / 影棚实景冲突 / 版型冲突三类告警与配方 ID 串留档
7. **服饰套图批量生成**（注入模块，`拼装库工程/integrate_batch.py` 同步）— 上传**固定模特面部照＋模特全身照**与**多张服装参考图**（每张一套），选风格与场地后，每套输出**同一背景**的 2 张全身（不同动作）＋ 1 张半身（随机动作）；动作全部抽取自模板库（风格动作＋通用动作，半身只用兼容半身动作，影棚自动剔除道具依赖动作，备注标「已弃用」的动作自动出池——G-动作2/4/7 已于 2026-07-16 实测弃用）。提示词内置四重锁定：参考图角色声明＋冲突裁决（**参考图＞文字**）、服装 100% 保真、模特人脸＋身材锁定、整组同景锁定，并强制**面部与头上零饰品**（参考图佩戴也去除）；每张图可复制真实提示词、可一键导出全部任务提示词直接投喂 Nano Banana Pro。**2026-07-19 模块库扩容**（475→506 条）：通用动作新增 G-动作13~27（宴会持包/背身回眸/走姿等，全部自动入池）、「表情」新增 温柔亲和/端庄平静/沉静内敛/慵懒疏离/清冷沉静 五档（仍全档恒定禁笑）、「发型」新增侧分低发髻、新增「妆容」档位（新中式红唇/冷艳锐利，只管妆面、可与发型表情自由叠加）、实景场景新增酒店系列 G-场景-酒店1~7（大堂光斑/旋转楼梯/走廊电梯厅/挑高明堂/宴会红墙）

素材以 **chip 芯片** 形式进入输入框，生成上下文所见即所得、可随时移除。

## 真实生成接入（APImart，2026-07-16）

已内置 APImart 网关直连（浏览器直接调用，无需服务端；协议：`POST /v1/images/generations` → 轮询 `GET /v1/tasks/{id}`）。**配置 Key 后自动从模拟演示切换为真实出图**，未配置时一切照旧本地模拟。

- **生图模型三选一**（输入框右下角选择器，批量弹窗内也可切换）：
  | 模型 | ID | 说明 |
  |---|---|---|
  | Nano Banana Pro | `gemini-3-pro-image-preview` | 参考图×14 · 最高4K · 服饰保真主力（默认） |
  | GPT-Image 2 | `gpt-image-2` | 参考图×16 · 最高4K |
  | Seedream 5.0 Pro | `doubao-seedream-5-0-pro` | 参考图×10 · 最高2K · 质量高较慢 |
- **Key 配置两种方式**：① 工具栏「⚙ 生成服务设置」粘贴 Key（存本机 localStorage，可测连接看余额）；② 在 index.html 同目录放 `apiconfig.local.js`（模板见设置弹窗说明，**已被 .gitignore 排除**）。
- **⚠ 安全红线**：本仓库是公开 GitHub Pages——Key 绝不能写进 index.html，也绝不能提交 `apiconfig.local.js`；线上页面无 Key，自动回落为演示模式。
- **对话创作流**：真实模式一次出 2 张（每张独立任务、独立进度，失败可单张重试），上传图 / 资产图 / 场景图作为参考图随任务发送；比例与清晰度即工具栏所选。
- **批量套图**：真实模式默认「**首图锁场景**」——每套先出全身A，成图自动作为场景参考图（提示词自动插入 `G-锁定-场景参考1`）传给全身B与半身，同背景一致性最强；可切回「句子同景」。**拍摄背景三种玩法**：① 实景——该风格全部场景模块可选（场景1~4 + 通用棚拍场景 G-场景-棚拍1~6 + 酒店实景 G-场景-酒店1~7）；② 影棚——背景色可选（G-棚色1~8：纯白/浅灰/中性灰/深炭灰/纯黑/奶油米/暖灰/莫兰迪灰蓝，白衣提示换灰底）；③ **自定义背景图**——上传一张背景照或从「我的场景/公共场景」库点选，作为第 4 张参考图（`G-锁定-场景参考1` 声明句）传给模型，三张图同用这张背景、背景/色调/光线以照片为准，此模式下风格场景句/光线句/同景句/首图锁场景自动退出（背景已被照片钉死，天然同景）。**服装零添加**：每条提示词恒带 `G-约束-服装无添加1`——参考图中不存在的部件（内搭/腰带/口袋/花边/图案等）一律不允许出现，衣长袖长领型不改，与保真锁定（不改已有细节）互补。**表情默认酷感锐利**：「表情」选择器默认拼入 `G-表情-酷感1`（眼神锐利、冷冽克制、不带笑意的时装大片气场，拼在动作句后并显式覆盖动作模板自带的笑意——只保留动作姿势与视线方向），可切回「跟随动作描述」。**耳饰按服装自动搭配**（`G-耳饰-细巧1`/`G-耳饰-华丽1` + 露耳版无饰约束 `G-约束-头面无饰2`）：「耳饰」默认自动——旗袍款配细巧耳线（若隐若现）、晚礼服风格(S18)配华丽水晶长坠、其余不戴；开耳饰时头面无饰约束自动切换为露耳版（除耳饰外其余照禁），可手动锁定 无/细巧/华丽。**身材默认轻微优化**（`G-身材-微调1`，参考图身材的修正层）：手臂细一丁点、腿长一点点、臀部稍微饱满一点点防扁平——幅度均极小、不变形，可切回严格按参考图。**表情默认骄傲张扬**（`G-表情-骄傲1`）：下巴微扬带出鼻尖一点点上翘的傲气、眼神张扬自信、嘴角似有若无上挑，明确禁苦相（不皱眉/不撇嘴/不垂眼），不改鼻型五官；酷感锐利与跟随动作仍可选。**所有表情档位恒定禁笑**（`G-表情-禁笑1` 硬约束）：不微笑/不浅笑/不露齿/不咧嘴/不眯眼含笑，动作模板句自带的笑容也会被压制，嘴角至多保持似有若无的克制弧度。**发型默认中分低盘发**（`G-发型-低盘1`，按用户参考款固化）：中分向后光洁梳拢＋脑后低发髻＋露耳露颈＋仅鬓角微碎发、无发饰；只锁款式、发色仍跟随模特面部参考图，可在「发型」切回跟随参考图。**姿态默认松弛补强**（`G-姿态-松弛1`，治动作僵硬）：在保持动作姿势与构图的前提下追加执行层修饰——压重心、松肩、肘指带自然弧度、抓拍中间帧、衣摆发丝微动态，不改写动作本身，可在「姿态」切回严格按动作句。**鞋子默认裸色**（`G-鞋-裸色1`，仅全身拼接）：全身图鞋子统一裸色/浅杏色简约款、显腿长，无论参考图有没有鞋都按此呈现，可在「鞋子」切回不指定。**旗袍款支持**（`G-版型-旗袍及踝1`）：「款式」选「旗袍款」锁定裙长及至脚踝（开衩再高裙长不变，立领/盘扣/开衩按参考图），刻意做成选择器而非常驻句、防止把非旗袍服装带偏。**比例与清晰度可在弹窗内自由选择**：比例默认「跟随风格推荐」（全身/半身可不同画幅，如晚礼服 9:16 / 3:4），也可强制 3:4 / 1:1 / 9:16 / 16:9（提示词开头的画幅短语随之同步替换）；清晰度 1K/2K/4K 默认跟随全局选择器。**并发与多次生成**：并发数可选（顺序 / ×2 / ×4 / ×6，跨套并行调度，首图锁场景的套内依赖自动保持：A 出完才放行 B 与半身，不同套之间照常并行）；「组数」支持每套 ×1/×2/×3 组多次生成（第 2 组起动作重新随机抽取），成功的图也可点「重新生成」单张重出变体。**支持多次点击叠加批次**：生成按钮不锁定，点完弹窗保持打开，调整参数（换服装/风格/棚色/比例均可）即可再次点击叠加下一批——每批参数与提示词在点击瞬间快照（互不干扰），所有批次共用一个全局并发池（上限=最近一批所选并发数），聊天流按「第 N 批」分组展示，导出按钮导出的是对应批次的快照提示词。所有卡片先建骨架屏再由调度池填充、逐格进度、失败单格重试，任务数超过 9 前有额度确认。
- **下载与打包**（2026-07-16）：设置弹窗可设「**下载位置**」——File System Access 选定文件夹一次（句柄存 IndexedDB，重启记住），之后所有下载与打包**静默直存**该文件夹；未设置时走「另存为」（每次自选位置），旧浏览器退回默认下载。批量结果卡有「**打包下载本批**」：能抓到字节的图统一转 JPG 打成 **ZIP**（零依赖 STORE 打包器，内含 `清单.txt` 文件名↔套/组/动作对照）；APImart 图床（getapib.org）**未开放 CORS**、浏览器拿不到真实图字节时，自动降级生成 **curl 一键下载脚本 .bat**（纯 ASCII，Windows 自带 curl，双击即把整批图存到脚本所在文件夹）。单张「下载」同样走该链路，跨域拿不到时新标签打开可右键另存。文件名规则 `p批_t套_g组_fullA|fullB|half.jpg`。
- **刷新不丢**（2026-07-16）：生成素材本地持久化（IndexedDB）——每张生成图（对话流＋批量，含完整提示词与模型/画幅信息）自动记入**生成历史**（上限 300 条），刷新/重开页面后聊天区自动出现「已恢复 N 条生成记录」卡片（最近 24 张成网格，可复制提示词/下载/存资产/放大，可一键清空）；**资产库与我的场景**持久保存（上传图已从 blob 改为 dataURL，刷新不失效）；**批量弹窗素材**（模特面部照/全身照/服装列表/背景图）自动保存，刷新后打开弹窗即恢复、无需重传（动作按当前风格自动重抽）。
- **生成链接 24 小时过期**：请及时「下载 / 打包下载」或「存入资产库」。
